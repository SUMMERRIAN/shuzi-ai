# 树子AI界面风格规范

更新时间：2026-06-06

这份文档记录 2026-06-05 对树子AI整体 UI 进行统一美化后形成的界面规范。后续新增页面、修改页面、优化组件时，优先遵守这里的风格，避免不同页面出现割裂感。

## 总体气质

树子AI不是营销型网站，也不是花哨的工具站。整体界面应该像一个安静、可信、长期陪伴学生学习的个人学习教练系统。

关键词：

- 安静
- 清楚
- 可信
- 有秩序
- 适合学生和家长长期使用
- 不做夸张装饰，不做强烈科技感

页面第一感觉应该是：干净、温和、专业，但不冰冷。

## 字体与字重

当前字体策略：

- 使用系统无衬线字体，不额外引入复杂字体包。
- 中文优先保证清晰可读，避免过度装饰。
- 标题可以有力量，但不能过黑、过粗。
- 导航、按钮、卡片标题使用中等偏粗，不使用特别厚重的黑体效果。

建议字重：

- 页面大标题：`font-weight: 780-820`
- 卡片标题：`font-weight: 700-760`
- 导航文字：`font-weight: 650-700`
- 按钮文字：`font-weight: 700`
- 正文：`font-weight: 400-500`
- 辅助说明：`font-weight: 400`

避免：

- 大面积使用 `900` 粗体。
- 侧边栏全部粗黑，容易压迫。
- 小卡片内使用超大字号。
- 为了突出而让所有标题都很重。

## 字号层级

页面要有清晰层级，但不要到处都是大字。

建议：

- 首页主标题可以最大，体现品牌和网站定位。
- 功能页标题保持克制，通常 26-36px。
- 卡片标题通常 16-20px。
- 表格、记录、说明文字保持 13-15px。
- 按钮文字通常 14-16px。

原则：

- 功能页面是给学生反复使用的，不做“巨型宣传页”。
- 管理员后台、会员中心、学习计划、学情画像等页面要偏工作台风格。
- 重要信息靠布局和留白突出，不只靠放大字号。

## 颜色体系

主色调：

- 深色文字：接近黑色，但不纯黑压死，当前偏 `#111820`。
- 主按钮：黑色或近黑色，形成稳定、专业的操作感。
- 强调色：树子AI绿色，当前偏 `#008653`。
- 背景：温暖的浅米白/浅灰白，不使用纯冷白铺满。
- 边框：低饱和灰绿、米灰色，当前偏 `#d8d8cf`、`#dfe5da`。

视觉感觉：

- 背景不要一片纯白。
- 卡片与页面背景要有轻微区分。
- 绿色只用于状态、标签、轻强调，不让全站变成单一绿色。
- 黑色按钮用于主要动作，绿色更多用于标识和辅助。

避免：

- 大面积纯白，页面会显得空、散。
- 大面积绿色，容易变成单色主题。
- 强烈渐变、霓虹色、蓝紫科技风。
- 过度阴影和漂浮卡片。

## 背景与卡片

当前整体背景采用温暖浅底，卡片使用暖白或轻微米白。

卡片规则：

- 卡片圆角保持克制，通常 `8-10px`。
- 卡片边框要可见但轻，形成内容边界。
- 阴影要非常轻，只用于增加层次。
- 不要把卡片套卡片套得很深。

适合使用卡片的地方：

- 重复内容项
- 管理后台模块
- 学情画像分项
- 任务卡
- 记录列表
- 弹窗

不适合过度卡片化的地方：

- 整个页面区块不要全都像漂浮广告卡。
- 表单内部不要过度嵌套小卡片。

## 按钮风格

昨天确定的按钮风格偏参考图中的简洁黑白按钮：

- 主按钮：黑底白字，圆角约 10px。
- 次按钮：白底、深色边框、深色文字。
- 危险操作：单独使用红色或弱红色，不混入主按钮。
- 按钮不要过高，不要过圆。

按钮状态：

- hover 可以轻微上移或加浅阴影。
- disabled 要明显变淡。
- 主要动作按钮保持稳定，不使用跳动、闪烁、强动画。

适用：

- AI生成
- 保存
- 确认入账
- 登录
- 下载
- 新建/刷新

## 侧边栏

侧边栏是全站主要导航，风格要稳定。

当前方向：

- 深色背景。
- 当前选中项使用浅色块。
- 导航文字不要太粗，避免压迫。
- 图标与文字间距保持均匀。
- 会员状态卡可以使用深绿色弱透明背景，但不要太亮。

侧边栏文字建议：

- 不用超粗体。
- 当前页面可以略加粗。
- 普通项保持中等字重。

## 表单与输入框

表单是学生长期填写的核心界面，必须舒服。

规则：

- 输入框背景以白色为主。
- 边框浅灰绿。
- 圆角 8px 左右。
- 文本框留足内边距。
- placeholder 要清楚，但颜色不能太重。

不要：

- 输入框边框太黑。
- 一堆控件没有分组。
- 按钮离输入区域太远。
- 清空、删除类按钮过于显眼。

## 数据和后台页面

管理员后台、会员中心、AI成本对账本属于操作型页面。

设计原则：

- 信息密度可以比学生页面高一点。
- 使用清晰表格、统计卡、分组卡。
- 标题要短，说明要直接。
- 关键数字要醒目，但不夸张。
- 保持黑色主按钮和暖白卡片风格。

AI成本对账页的样式方向：

- 顶部说明卡。
- 统计总览卡片。
- 供应商拆分卡片。
- 功能平均成本表。
- 每日汇总和调用明细列表。
- 明细区域允许滚动，避免页面无限变长。

## 首页风格

首页负责让学生和家长快速知道网站是做什么的。

首页结构方向：

- 顶部整体介绍。
- 学习闭环流程说明。
- 每个功能页面用折叠/下拉方式介绍。
- 最后说明整个网站的学习辅助逻辑。

首页不需要大量按钮，不做营销感强的落地页。

首页应回答：

- 树子AI是什么？
- 对学生有什么用？
- 对家长有什么用？
- 每个页面解决什么问题？
- AI如何参与学习辅助？

## 页面之间的一致性

后续所有页面都要保持：

- 同样的背景色。
- 同样的卡片圆角和边框。
- 同样的按钮风格。
- 同样的标题字重。
- 同样的侧边栏导航风格。
- 同样的标签样式。

如果新增页面，先检查：

- 是否使用了 `hero-band` 或相近页面头部。
- 是否使用了 `panel` 或相近容器。
- 主按钮是否用黑色风格。
- 是否避免了过粗字体。
- 卡片和背景是否有明显但柔和的区分。

## 特别注意

后续如果只是 UI 修改：

- 不改 AI 提示词。
- 不改 API 行为。
- 不改会员扣费逻辑。
- 不改数据保存逻辑。
- 不改已经调好的图片处理、生图、错题专项、没有答案等核心流程。

只有用户明确要求修改业务逻辑时，才进入逻辑代码。

## 快速检查清单

每次 UI 修改完成后检查：

- 字体是否过粗？
- 页面是否一片纯白？
- 卡片和背景是否有区分？
- 主按钮是否统一？
- 侧边栏是否压迫？
- 移动端是否不会挤坏？
- 文本是否清楚易懂？
- 有没有误改业务逻辑？

这份文档作为树子AI后续 UI 修改的基准。
