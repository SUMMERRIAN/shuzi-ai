# 树子AI网站 MVP 说明

## 当前版本

当前已完成一个可运行的前端 MVP，适合用于产品演示、页面确认和第一轮业务流程打磨。

当前前三页已经按“完整学习通路”重构：

- 第一页：学习问题分析。包含学习问卷、文字主诉、语音主诉、个人档案、学习画像、OpenAI 智能体标准输出格式。
- 第二页：学习策略与任务制定。根据第一页画像，为语文、数学、英语、物理等科目生成策略，并加入学习任务。
- 第三页：学习计划制定。先收集孩子周一到周日时间规律，再把第二页任务排入空闲时间，形成可打印/可下载计划模板。

其中，第一页问卷参考了 `自助学习·学生基本信息手册水印版.pdf` 的资料采集方向，以及 `夏雨学习法测评报告-发送给家长的版本.pdf` 的学情诊断维度。

## 本地运行

```bash
npm install
npm run dev
```

构建静态文件：

```bash
npm run build
```

构建后的文件在 `dist/` 文件夹。

## Hostinger 部署建议

如果你现在购买的是 Hostinger 普通虚拟主机，可以先把 `dist/` 里的内容上传到 `public_html`，作为第一版展示与演示网站。

如果要做正式会员系统，需要后端服务与数据库。推荐两种路线：

1. Hostinger VPS / Node.js 路线：React 前端 + Node.js API + PostgreSQL/MySQL。
2. Hostinger 普通虚拟主机路线：React 前端 + PHP API + MySQL。

正式商业版需要增加：

- 用户登录注册、手机号/微信登录、密码找回。
- 学生、家长、老师、管理员四类角色权限。
- 学生档案、测评记录、学习计划、任务反馈、错题图片、AI 报告的长期存储。
- 文件上传与图片 OCR，识别试卷、题干、答案、错因。
- AI 接口，用于学习问题分析、学习画像生成、计划生成、错题变式训练。
- 支付与会员到期管理。
- 数据备份、隐私协议、访问日志和内容审核。

## OpenAI 后端接口建议

正式版不要在前端直接放 OpenAI API Key。建议由后端提供以下接口：

- `POST /api/students/:id/intake`：保存学习问卷。
- `POST /api/students/:id/complaints`：保存孩子文字主诉。
- `POST /api/students/:id/voice`：上传语音，转写后进入个人档案。
- `POST /api/students/:id/profile/generate`：调用 OpenAI 智能体，生成学习画像报告。
- `POST /api/students/:id/strategies/generate`：根据画像生成科目策略和方法训练任务。
- `POST /api/students/:id/plans/generate`：根据任务和空闲时间生成周学习计划。
- `GET /api/students/:id/questionnaire.pdf`：后台生成问卷 PDF。
- `GET /api/students/:id/study-plan.pdf`：后台生成学习计划 PDF。

智能体输出建议固定为 JSON，便于保存和渲染：

## 建议的数据库表

- `users`：用户账号、角色、联系方式、会员状态。
- `students`：学生基本信息、年级、目标、学校、家长绑定。
- `student_profiles`：学习画像维度分数、画像版本、生成时间。
- `diagnosis_records`：学习问题输入、AI 诊断结果、老师修订意见。
- `study_plans`：周计划、日计划、任务模板。
- `task_logs`：任务完成情况、质量反馈、未完成原因。
- `mistake_files`：上传的试卷、错题、作业图片。
- `mistake_questions`：题干、知识点、错因、解析、训练状态。
- `generated_practice`：AI 生成的同类题、答案、训练结果。
- `reports`：周报、月报、家长报告。

6. 做会员支付、家长端、老师后台和部署上线。
