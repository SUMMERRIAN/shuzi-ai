# 注册问题排查记录

更新时间：2026-06-07

## 背景

用户反馈：欧洲马德里访问和注册正常，但中国大陆朋友反馈可以顺利打开网站，注册不能成功。

本次只做排查记录，不修改代码。

## 当前代码链路

前端注册入口：

- `src/apiClient.js` 默认使用 `/api` 作为后端接口前缀。
- `src/main.jsx` 的注册逻辑调用 `/auth/register`。

后端注册接口：

- `server/index.js` 中的 `POST /api/auth/register`。
- 注册流程为：用户名规范化、密码长度校验、查 PostgreSQL 用户是否已存在、bcrypt 加密密码、写入用户、创建学生与账号相关行、返回 JWT。

关键判断：

- 注册接口不调用 OpenAI。
- 注册接口不调用 Gemini。
- 注册接口不依赖图片生成。
- 注册逻辑本身没有地区判断。

## 已做线上检查

从当前网络环境测试：

- `https://xyshuziai.com/api/health` 正常返回。
- `https://www.xyshuziai.com/api/health` 正常返回。
- `/api/auth/register` 能正常打到后端，并能返回后端校验错误，例如用户名不足 3 个字符时返回 `USERNAME_REQUIRED`。

说明：

- 线上 API 服务当前是活的。
- nginx `/api` 代理当前可用。
- 注册 POST 请求至少从当前线路可以到达后端。

## 初步判断

如果中国大陆用户能打开网页但注册失败，更可能是以下问题，而不是注册代码本身：

1. 中国大陆到动态 API 请求 `/api/auth/register` 的线路不稳定。
2. 使用微信内置浏览器、第三方浏览器或缓存环境导致请求异常。
3. 用户实际访问入口不一致，例如使用了 `www`、HTTP、中间跳转页、旧缓存地址等。
4. Cloudflare、DNS、HTTPS 或动态接口路由在中国大陆部分网络环境下不稳定。
5. 用户点击的可能是登录而不是注册，或用户名已存在、密码格式不符合要求，需要看具体错误提示。

## 后续排查步骤

让中国大陆朋友先做两个最小测试：

1. 直接打开：

```text
https://xyshuziai.com/api/health
```

正常应看到：

```json
{"ok":true,"service":"shuzi-ai-api"}
```

2. 使用系统浏览器访问网站，注册一个全新的用户名，并截图失败提示。

同时可在服务器查看后端日志：

```bash
cd /var/www/shuzi-ai
journalctl -u shuzi-ai-api --since "1 hour ago" --no-pager | tail -200
```

判断方式：

- 如果用户点击注册时，服务器日志没有任何对应请求，优先排查大陆网络、浏览器、Cloudflare、DNS 或 nginx 入口。
- 如果服务器日志有请求并返回具体错误，再根据错误码判断是否需要修改后端或前端提示。

## 暂不建议修改的原因

目前没有证据表明注册代码本身有错误。贸然修改可能会影响已经正常工作的欧洲线路和现有会员系统。

建议先收集中国大陆用户的：

- 访问 URL。
- 浏览器类型。
- 注册失败截图。
- `/api/health` 测试结果。
- 对应时间段服务器日志。

拿到这些信息后，再决定是否修改 CORS、域名跳转、Cloudflare 设置、前端错误提示或后端注册逻辑。
