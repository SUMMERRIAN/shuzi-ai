# 树子AI数据备份与恢复说明

这份说明用于保护正式客户数据。代码已经在 GitHub，真正需要优先备份的是 PostgreSQL 数据库和服务器上传文件。

## 备份内容

1. PostgreSQL 数据库

数据库里包含用户账号、密码哈希、会员状态、学习币余额和交易记录、学情问卷、学情陈述、学情画像、学习任务、学习计划、错题、知识笔记、AI 自由问历史、学习社区等核心数据。

2. 上传文件目录

服务器默认上传目录是：

```bash
/var/www/shuzi-ai/uploads
```

这里包含用户上传的图片、PDF、音频、资料库文件等。数据库里保存的是文件记录和路径，真实文件仍然需要单独备份。

## 一键备份

在本地 PowerShell 进入项目目录后执行：

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\backup-shuzi-ai.ps1"
```

脚本会自动：

1. 连接服务器 `root@5.223.91.77`
2. 在服务器生成 PostgreSQL 数据库备份
3. 打包 `/var/www/shuzi-ai/uploads`
4. 下载到本地备份目录
5. 生成 SHA256 校验文件

默认会优先保存到 Google Drive 目录，例如：

```text
I:\Google Drive\ShuziAI-Backups
```

如果没有找到 Google Drive 目录，会保存到桌面：

```text
C:\Users\Administrator\Desktop\ShuziAI-Backups
```

也可以手动指定保存位置：

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\backup-shuzi-ai.ps1" -BackupRoot "I:\Google Drive\ShuziAI-Backups"
```

## 备份频率建议

- 数据库：每天至少一次
- 上传文件：前期可以每周一次；如果客户上传资料较多，改为每天一次
- 大版本更新前：手动备份一次
- 本地保留：最近 7 天每日备份、最近 4 周每周备份、最近 3 个月每月备份

脚本会自动清理服务器上超过 7 天的临时备份，避免服务器磁盘被备份文件占满。本地 Google Drive 里的备份不会自动删除。

## 恢复思路

恢复时需要同时恢复两部分：

1. PostgreSQL 数据库 `.dump`
2. 上传文件压缩包 `.tar.gz`

恢复前必须先停止服务：

```bash
systemctl stop shuzi-ai-api
```

再恢复数据库和上传文件。恢复完成后重启服务：

```bash
systemctl restart shuzi-ai-api
systemctl reload nginx
```

正式恢复命令需要根据当时服务器状态决定，不能随意覆盖生产库。恢复前应先确认备份文件日期、数据库名、上传目录和当前服务器情况。

## 安全提醒

备份文件里包含客户隐私、会员和学习币记录，必须放在私密目录。不要把备份文件发到公开群、公开网盘或 GitHub。
